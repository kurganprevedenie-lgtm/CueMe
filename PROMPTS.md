# Промпты CueMe

Все промпты, которыми бот зовёт LLM (Groq, llama-3.3-70b). Скопируй в другую
нейронку с задачей улучшить стиль/качество вывода.

Плейсхолдеры в `{фигурных скобках}` подставляются в рантайме:
- `{style_card}` — голос автора (как пишет сам пользователь)
- `{interaction_card}` — наблюдения о собеседнике (как с ним эффективнее писать)
- `{draft}` — черновик пользователя
- `{incoming}` — сообщение собеседника
- `{...sample}` / `{...msgs}` — выборка реальных сообщений из переписки
- `{features_summary}` / `{stats}` — числовая статистика (длина, эмодзи, инициатива и т.п.)

Источник: [llm.py](llm.py).

# ЧАСТЬ 1 — Генерация сообщений для пользователя

Эти три промпта формируют **текст, который видит пользователь**. Главные кандидаты на доработку стиля.

## 1.1 Переписать черновик (`rewrite_message_explained`)

```
Ты переписываешь сообщение от лица конкретного человека — под конкретного собеседника.

ГОЛОС АВТОРА (style_card):
{style_card}

СОБЕСЕДНИК — его привычки и что у него заходит (interaction_card):
{interaction_card}

ЧЕРНОВИК АВТОРА:
{draft}

=== ЧТО СОХРАНЯЕМ ===
• Личный словарный запас, смысл, регистр. Эмодзи не добавляй если их нет.
• Длину — держи ПРИМЕРНО как в черновике, не сокращай резко.
=== ЧТО МЕНЯЕМ под собеседника ===
• Тон, первую фразу, формальность, ритм.

Сначала выведи ТОЛЬКО переписанное сообщение — без кавычек.
Затем на отдельной строке: ===ПОЯСНЕНИЕ===
Затем 1-2 коротких фразы что именно ты изменил под этого собеседника
(например: «сократил вдвое, убрал формальность, заход сделал теплее»).
```

> Есть упрощённая версия `rewrite_message` (без пояснения) — используется в авто-режиме.
> У неё дополнительно даны примеры глубины переписывания:
> ```
> ПРИМЕРЫ ожидаемой глубины:
> Черновик: «хотел спросить, ты не забыл про встречу в пятницу?»
> → Теплее: «слушай, ты же помнишь про встречу в пятницу?»
> → Формально: «напоминаю про нашу встречу в пятницу, всё в силе?»
> ```

## 1.2 Ответить за пользователя (`suggest_reply`)

```
Собеседник прислал автору сообщение. Предложи КАК ответить ему — в голосе автора,
с учётом привычек собеседника.

ГОЛОС АВТОРА (style_card):
{style_card}

СОБЕСЕДНИК (interaction_card):
{interaction_card}

СООБЩЕНИЕ СОБЕСЕДНИКА:
{incoming}

ПРАВИЛА:
• Ответ в стиле автора: его слова, регистр, длина под собеседника
• Не выдумывай факты, которых автор знать не может
• Если по сообщению нужна конкретика которой нет — предложи короткий уточняющий ответ
• Это черновик ответа от лица автора, а не совет со стороны

Сначала выведи ТОЛЬКО текст ответа — без кавычек.
Затем на отдельной строке: ===ПОЯСНЕНИЕ===
Затем 1 короткую фразу почему ответ такой.
```

## 1.3 Подстройка тона (`adjust_message`) — кнопки короче/теплее/формальнее

```
Подправь готовое сообщение по инструкции, сохранив голос автора и смысл.

ГОЛОС АВТОРА (style_card):
{style_card}

ТЕКУЩЕЕ СООБЩЕНИЕ:
{current_text}

ИНСТРУКЦИЯ: {instruction}

ПРАВИЛА:
• Сохрани смысл и личный словарь автора
• Регистр (большая/маленькая буква) и эмодзи — как в исходном, если инструкция не про них

Сначала выведи ТОЛЬКО изменённое сообщение — без кавычек.
Затем на отдельной строке: ===ПОЯСНЕНИЕ===
Затем 1 короткую фразу что изменилось.
```

Варианты `{instruction}`:
- **короче** → `Сделай ЗАМЕТНО короче — выкинь лишнее, оставь суть.`
- **теплее** → `Сделай теплее и мягче по тону, дружелюбнее.`
- **формальнее** → `Сделай формальнее и вежливее, более деловой регистр.`

# ЧАСТЬ 2 — Аналитические промпты (строят «карточки»)

Эти промпты вызываются при импорте чата / авто-пересборке. Их вывод (`style_card`,
`interaction_card`) потом подставляется в промпты из Части 1.

## 2.1 Голос автора (`build_style_card`)

```
Проанализируй сообщения пользователя.
Верни ТОЛЬКО текст анализа — без JSON, без кавычек, без markdown.
Просто текст с заголовками секций и пунктами через •.

СТАТИСТИКА:
{user_features_summary}

ВСЕГО СООБЩЕНИЙ В АНАЛИЗЕ: {N}

СООБЩЕНИЯ ПОЛЬЗОВАТЕЛЯ:
- ...

ПРАВИЛА (строго):
• Только конкретные факты из сообщений — никаких общих слов
• Каждый пункт: наблюдение + пример-цитата
• Статистика — первичный источник. Если emoji/сообщ < 0.3 → «эмодзи почти не использует»
• Регистр — обязательный пункт: пишет с большой или маленькой — проверь
• Запрещено: «общительный», «тёплый», «использует юмор» — без конкретики

ФОРМАТ (секции разделены пустой строкой):

ГОЛОС И ТОН
• [факт + пример-цитата]

СТРУКТУРА СООБЩЕНИЙ
• [типичная длина в словах + пример]
• [пунктуация, абзацы]

СЛОВАРНЫЙ ЗАПАС
• [характерные слова — цитаты]

ЮМОР И ЭМОЦИИ
• [как выражает, с примером]

РЕГИСТР И ИНИЦИАТИВА
• [с большой или маленькой — факт]
• [кто начинает темы]
```

## 2.2 Наблюдения о собеседнике (`build_interaction_card`)

```
Проанализируй как СОБЕСЕДНИК общается в этой переписке.
Верни ТОЛЬКО текст анализа — без JSON, без кавычек, без markdown.
Просто текст с заголовками секций и пунктами через •.

СТАТИСТИКА:
{features_summary}

ВСЕГО В АНАЛИЗЕ: мои — {N}, собеседника — {M}

СООБЩЕНИЯ ПОЛЬЗОВАТЕЛЯ:
- ...
СООБЩЕНИЯ СОБЕСЕДНИКА:
- ...

ПРАВИЛА (строго):
• Конкретные наблюдения + пример-цитата из сообщений
• ЗАПРЕЩЕНЫ общие слова без примеров: «общается дружелюбно», «любит юмор»
• Статистика — первичный источник для эмодзи и длины
• Это наблюдения о том КАК человек общается — НЕ психологический портрет

ФОРМАТ (секции разделены пустой строкой):

ДЛИНА СООБЩЕНИЙ
• Типичный размер в словах (диапазон) + пример из переписки

ТЕМП И РИТМ
• Характерное время ответа, паузы

РЕГИСТР И ЯЗЫК
• Ты/Вы, с большой или маленькой — проверь в сообщениях
• Характерный сленг (цитаты)
• Эмодзи: взять цифру из статистики, не придумывать

ИНИЦИАТИВА
• Кто чаще пишет первым; задаёт ли встречные вопросы

ЧТО ДАЁТ ЖИВОЙ ОТКЛИК
• Конкретный пример из переписки когда ответил развёрнуто

ЧТО НЕ РАБОТАЕТ
• Конкретный пример когда ответ был сухим или его не было

КАК ПИСАТЬ ЭТОМУ ЧЕЛОВЕКУ
• [3-4 конкретных практических совета]
```

## 2.3 Как автор пишет конкретному человеку (`build_my_style_for_contact`)

```
Проанализируй как пользователь пишет ЭТОМУ конкретному человеку.
Верни ТОЛЬКО текст — без JSON, без кавычек, без markdown.

СТАТИСТИКА:
{stats_summary}

ВСЕГО СООБЩЕНИЙ В АНАЛИЗЕ: {N}

МОИ СООБЩЕНИЯ К ЭТОМУ ЧЕЛОВЕКУ:
- ...

ПРАВИЛА:
• Анализируй ТОЛЬКО мои сообщения — не собеседника
• Конкретные наблюдения + цитата из текста
• Что специфично именно для этой переписки
• Регистр обязательно: с большой или маленькой буквы
• Запрещено: общие слова без цитат

ФОРМАТ:

ТОН И ДИСТАНЦИЯ
• [наблюдение + цитата]

ДЛИНА И СТРУКТУРА
• [типичная длина в словах + пример]

ФОРМАЛЬНОСТЬ И РЕГИСТР
• [ты/Вы, с большой или маленькой]

ЗАХОДЫ И ПЕРЕХОДЫ
• [как начинаю, как заканчиваю — цитаты]

ЧТО ХАРАКТЕРНО ИМЕННО ДЛЯ ЭТОЙ ПЕРЕПИСКИ
• [что может отличаться от других чатов]
```

## 2.4 Общий портрет — агрегат (`build_overall_style`)

```
Ниже — описания стиля одного человека с разными собеседниками (анонимизированы).
Найди паттерны. Верни ТОЛЬКО текст — без JSON, без кавычек, без markdown.

СТИЛИ ПО СОБЕСЕДНИКАМ:
Собеседник 1:
...
Собеседник 2:
...

ПРАВИЛА:
• Ищи что МЕНЯЕТСЯ и что ОСТАЁТСЯ постоянным
• Конкретные наблюдения — ссылайся на примеры из карточек
• НЕ пересказывай каждую карточку — анализируй паттерны
• Запрещено: «адаптируется к собеседнику» без конкретики

ФОРМАТ:

ЧТО ОСТАЁТСЯ ПОСТОЯННЫМ
• [общее для всех переписок + пример]

КАК МЕНЯЕТСЯ СТИЛЬ
• [с кем формально vs неформально]
• [с кем длинно vs коротко]

ПАТТЕРНЫ АДАПТАЦИИ
• [как подстраивается под разных людей — конкретно]

СИЛЬНЫЕ СТОРОНЫ
• [что стабильно работает хорошо]
```

## 2.5 Сравнение стилей (`compare_my_styles`) — команда /compare

```
Ниже — как ОДИН человек пишет разным собеседникам. Сравни и покажи контраст между ними.
Верни ТОЛЬКО текст — без JSON, без кавычек, без markdown.

СТИЛИ ПО СОБЕСЕДНИКАМ:
=== {Имя 1} ===
...
=== {Имя 2} ===
...

ПРАВИЛА:
• По каждому собеседнику — 1-2 строки: чем именно отличается стиль с ним
• Опирайся на конкретику из карточек (длина, тон, формальность, эмодзи)
• В конце — короткий вывод: где ты теплее/суше, длиннее/короче, формальнее
• Имена собеседников бери как в заголовках

ФОРМАТ:

КАК ТЫ ПИШЕШЬ КАЖДОМУ
• [Имя]: [чем выделяется]

ОБЩИЙ ВЫВОД
• [с кем как — контраст в одну-две фразы]
```
