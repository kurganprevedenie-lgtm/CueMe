# Eval-харнесс промптов генерации

Инструмент для **измеримой** проверки качества ответов бота — чтобы каждую правку
промпта оценивать объективно, а не «на глаз». Реальные чаты не нужны: банк
синтетический.

## Состав

- `scenarios.py` — банк сценариев (rewrite/reply, тяжёлые кейсы: отказ, сухие
  ответы, уязвимость) + ожидаемые ограничения к ответу.
- `checks.py` — чистые детерминированные проверки (без LLM): зачин-клише,
  иностранный скрипт, ИИ-штампы, выпрашивание, длина. Покрыты юнит-тестами
  (`tests/test_eval_checks.py`).
- `judge.py` — LLM-судья: оценка ответа по 5-пунктной рубрике (балл 0-10 + вердикт).
- `run_eval.py` — раннер: гоняет банк через реальные функции генерации, применяет
  проверки, печатает скоркарту (доля без нарушений, разбивка по типам, топ зачинов).

## Запуск

```bash
# из корня репозитория
PYTHONPATH=. LLM_PROVIDER_ORDER=groq,openrouter \
  python eval/run_eval.py --out eval/eval_report.txt --pace 20
# c LLM-судьёй (в 2 раза больше вызовов):
PYTHONPATH=. python eval/run_eval.py --judge --pace 25
# быстрый прогон подвыборки:
PYTHONPATH=. python eval/run_eval.py --limit 6 --pace 20
```

## Про лимиты провайдера

Промпты крупные (~1800 токенов). Groq free-tier ограничивает токены в минуту И в
сутки — при полном прогоне (20 сценариев, тем более с `--judge`) легко упереться.
`--pace` разреживает вызовы (20-25с держат под RPM/TPM). При суточном лимите —
прогонять частями (`--limit`) или на провайдере с большей квотой (задать
`OPENROUTER_API_KEY` / `GEMINI_PROXY`).

## Как использовать для A/B

1. Прогнать на текущем промпте → зафиксировать скоркарту (доля без нарушений,
   средний балл судьи, топ зачинов).
2. Изменить промпт в `llm.py`.
3. Прогнать снова → сравнить. Регрессия видна по росту нарушений или падению балла.
